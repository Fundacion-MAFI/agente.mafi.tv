# Plan: Add `OPENAI_BASE_URL` support to OpenAI provider configuration

## Current Architecture

The project routes **all** OpenAI model calls through Vercel's AI Gateway (`@ai-sdk/gateway`):

- **`lib/ai/providers.ts`** — defines `myProvider` using `gateway.languageModel("openai/...")` for chat, title, and artifact models, and `filmAgentGateway` (with its own `AI_GATEWAY_FILM_AGENT_BASE_URL` override) for the film agent.
- **`lib/ai/embedding-models.ts`** — uses `gateway.textEmbeddingModel("openai/...")`.
- **No `@ai-sdk/openai`** package is installed — there is no direct OpenAI provider instance.
- **No `OPENAI_API_KEY`** or `OPENAI_BASE_URL` env vars exist yet.

## Approach

Introduce `@ai-sdk/openai` as a conditional provider. When `OPENAI_BASE_URL` is set, OpenAI models route through the custom server via the direct OpenAI provider. When it's absent, the existing Vercel AI Gateway behavior is preserved exactly.

This mirrors the existing `filmAgentGateway` pattern (conditional override with graceful fallback), but targets the main OpenAI models rather than just the film agent.

## Steps

### 1. Install `@ai-sdk/openai`

```bash
pnpm add @ai-sdk/openai
```

### 2. Update `lib/ai/providers.ts`

- Import `createOpenAI` from `@ai-sdk/openai`.
- Create a conditional OpenAI provider:

```ts
import { createOpenAI } from "@ai-sdk/openai";

const openaiBaseURL = process.env.OPENAI_BASE_URL?.trim();

const openaiProvider = openaiBaseURL
  ? createOpenAI({
      baseURL: openaiBaseURL,
      apiKey: process.env.OPENAI_API_KEY?.trim(),
    })
  : null;
```

- Update `myProvider` to use `openaiProvider` for OpenAI models when available, falling back to `gateway`:

```ts
: customProvider({
    languageModels: {
      "chat-model": openaiProvider
        ? openaiProvider("gpt-4o-mini")
        : gateway.languageModel("openai/gpt-4o-mini"),
      "film-agent": filmAgentGateway.languageModel("openai/gpt-4o"),
      "title-model": openaiProvider
        ? openaiProvider("gpt-4o-mini")
        : gateway.languageModel("openai/gpt-4o-mini"),
      "artifact-model": openaiProvider
        ? openaiProvider("gpt-4o-mini")
        : gateway.languageModel("openai/gpt-4o-mini"),
    },
  });
```

> **Note:** The `film-agent` model is intentionally left on `filmAgentGateway` since it already has its own `AI_GATEWAY_FILM_AGENT_BASE_URL` / `AI_GATEWAY_FILM_AGENT_API_KEY` override mechanism.

### 3. Update `lib/ai/embedding-models.ts` (optional)

Add the same conditional logic for embedding models so they also respect `OPENAI_BASE_URL`:

```ts
import { createOpenAI } from "@ai-sdk/openai";

const openaiBaseURL = process.env.OPENAI_BASE_URL?.trim();

const openaiProvider = openaiBaseURL
  ? createOpenAI({
      baseURL: openaiBaseURL,
      apiKey: process.env.OPENAI_API_KEY?.trim(),
    })
  : null;

export function getEmbeddingModel(modelId: EmbeddingModelId) {
  if (openaiProvider && modelId.startsWith("openai/")) {
    return openaiProvider.textEmbeddingModel(modelId.replace("openai/", ""));
  }
  return gateway.textEmbeddingModel(modelId);
}
```

### 4. Update `.env.example`

Add documentation for the new env vars:

```env
# Optional. Override the OpenAI API base URL for all OpenAI models (chat,
# embeddings, artifacts). Useful for proxies or OpenAI-compatible servers.
# When set, OPENAI_API_KEY is also required (bypasses the AI Gateway).
# OPENAI_BASE_URL=https://your-openai-proxy.example.com/v1
# OPENAI_API_KEY=sk-...
```

## Key Behaviors

- **`OPENAI_BASE_URL` not set** → All models route through Vercel AI Gateway as before (zero behavior change).
- **`OPENAI_BASE_URL` set** → OpenAI models route directly to the custom server, bypassing the AI Gateway. `OPENAI_API_KEY` is required in this case.
- **Film agent** → Unaffected; continues using its own `AI_GATEWAY_FILM_AGENT_*` config.
